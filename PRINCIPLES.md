# Principles

## Prime Directive

Every public note should be clear, restrained, secure, maintainable, and accurate.

## Operating Principles

- Public context should help people understand the work without exposing private systems.
- Documentation should describe what is true today, not what would sound impressive.
- Security boundaries should be explicit before a project grows.
- Maintenance is part of building.
- Agents should leave the ecosystem easier to understand than they found it.
- Small, well-described changes are preferred over broad undocumented rewrites.
- Private systems stay private even when they are mentioned publicly.
- Financial, betting, personal, and identity-related data require extra caution.

## Language Principles

Use:

- "early foundation"
- "planned"
- "needs review"
- "public-safe"
- "private / sensitive"
- "documented and true"

Avoid:

- "enterprise-grade"
- "fully secure"
- "audited"
- "guaranteed"
- "best"
- "revolutionary"
- "production-ready" unless the project is actually documented and operated that way.
