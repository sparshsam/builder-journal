# Ecosystem Map

This map describes the ecosystem at a high level. It is intentionally restrained so public readers can understand the direction without seeing sensitive details.

## Public-Safe Categories

| Category | Projects | Boundary |
|----------|----------|----------|
| Command center | Builder Journal | Public planning, status, and decisions only. |
| Standards | ecosystem-standards | Canonical rules for documentation, security, releases, and agent behavior. |
| Local documents | PDFReader | Public-safe document tooling; no private documents or file contents. |
| Proof and verification | OpenProof | No unsupported legal, compliance, or audit claims. |
| Care and household systems | OpenSprout, FridgeWise | No personal household data or private family details. |
| Behavioral finance | QuietLedger, Elora Vault, QuietVault | No personal financial records, account data, or unsupported security claims. |
| Focus and interfaces | StreamDock, LexiBoard, Chess by Sparsh, Hisstastic | Public descriptions should match actual maturity. |
| Private decision support | TW Oracle Personal Lab | High-level references only. No methodology, picks, prompts, architecture, data, or operational details. |

## Visibility Classes

| Class | Meaning | Public Handling |
|-------|---------|-----------------|
| Public | Safe to publish source, docs, issues, and releases. | Normal public repository standards apply. |
| Private | Contains proprietary or sensitive work. | Describe only purpose and boundary, if mentioned at all. |
| Sensitive | May involve personal, financial, operational, or security-relevant data. | Do not publish raw data or internal processes. |
| Experimental | Early idea, prototype, or lab. | Avoid launch claims, product claims, or guarantees. |

## Map Maintenance

Update this file when a project changes visibility, maturity, or ecosystem role. Keep changes concise and avoid turning the map into a private roadmap.
