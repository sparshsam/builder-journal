# Security

Builder Journal is a public repository. Security reporting and data-boundary expectations follow `ecosystem-standards`.

## Reporting

If you find a security issue in this repository, do not disclose it publicly in an issue. Contact the repository owner through GitHub and provide a concise description without posting secrets or exploit details in public.

## Boundaries

See `SECURITY_BOUNDARIES.md` for the repository-specific public/private and sensitive-data rules.

## Environment Files

Real environment files must never be committed. Use `.env.example` for placeholders only.
